来源：优化审计 Workflow `w9qzsjr7n`。
目标：按 P0 → P3 分阶段完成稳定性、正确性、开发质量、架构与运维改进。
跟踪约定：`[ ]` 未完成，`[x]` 已完成。

## 执行原则

- [ ] 每个 Task 完成后补充/更新测试。
- [ ] 每个阶段完成后运行 `pnpm lint`、`pnpm typecheck`、`pnpm test`。
- [ ] 涉及提交时，按项目记忆中的提交工作流执行：语义版本更新、推送所有 remote、多分支 pickup 合并。
- [ ] 涉及外部 API/队列/数据库语义变更时，先补回归测试再改实现。
