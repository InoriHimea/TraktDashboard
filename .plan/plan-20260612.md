来源：requirement.md 路线图 N1–N4（截至 2026-06-12）。
前置：plan-20260608（P0–P3）、episode-detail-page、tv-shows-and-movies-separation 均已完成。
目标：清偿遗留技术债、交付高价值功能、提升测试质量、补强基础设施。

## 执行原则

- [ ] 每个 Task 完成后补充/更新测试；全量测试（`pnpm test`）在阶段收尾前必须绿灯。
- [ ] 每个阶段完成后运行 `pnpm lint`、`pnpm typecheck`、`pnpm test:coverage`，门禁通过后方可进入下一阶段。
- [ ] 涉及 DB schema 变更时，先写 migration SQL，再更新 Drizzle schema，最后更新 TypeScript 类型。
- [ ] 涉及 API 新端点时，先定义类型（`packages/types`），再实现路由，最后实现前端 hook。
- [ ] 每个阶段完成后按提交工作流执行：语义版本升级 + 推送到两个远端。

## Phase 0 — N1：技术债清偿（遗留 `[~]` 项）

### [ ] N1-T01：将 HTTP 超时升级为真正的 AbortController 取消

优先级：N1（最高）
目标：消除 `Promise.race` 超时后请求仍在后台挂起的资源泄漏，让 Trakt/TMDB 请求可被真正中止。

- [ ] 在 `apps/api/src/lib/http.ts` 中将 `withTimeout` 重构为 `AbortController` 版本：创建 `controller`，`setTimeout` 后调用 `controller.abort()`，将 `signal` 传入 `fetch`；保持函数签名向后兼容。
- [ ] 将 `providerFetch`、`tmdbFetch`（`apps/api/src/services/tmdb.ts`）、`traktFetch`（`apps/api/src/services/trakt.ts`）中的 fetch 调用接入 `signal`。
- [ ] 确认 `AbortError` 被正确捕获并向上抛出可识别的超时错误（区别于网络错误）。
- [ ] 增加测试：模拟慢响应验证请求被 abort、abort 后不再 retry、`AbortError` 不计入 retry 次数。
- [ ] 影响文件：`apps/api/src/lib/http.ts`、`apps/api/src/services/trakt.ts`、`apps/api/src/services/tmdb.ts`。

### [ ] N1-T02：完成 Hono RPC hook 逐路由迁移

优先级：N1
目标：消除 `apps/web/src/hooks/index.ts` 中仍手写路径字符串的 `api.ts` 调用，切换为 typed `rpc` client，获得端到端类型安全。

- [ ] 审计 `apps/web/src/hooks/index.ts`，列出所有仍使用 `api.*` 而非 `rpc.*` 的 hook（预计涵盖 shows/movies/watchlist/calendar/stats/settings 等路由组）。
- [ ] 逐路由将 hook 内部的 `api.xxx()` 调用替换为 `rpc.api.xxx.$get()`/`$post()` 等 typed method（参考已迁移的示例）。
- [ ] 确保 `queryKeys.ts` factory 与新 hook 的 queryKey 保持一致，不引入 key 漂移。
- [ ] 迁移完成后，若 `apps/web/src/lib/api.ts` 中有路由完全无引用，移除对应方法，缩减手写 client 体积。
- [ ] `pnpm typecheck` 通过，所有 16 个 web 测试绿灯。
- [ ] 影响文件：`apps/web/src/hooks/index.ts`、`apps/web/src/lib/api.ts`、`apps/web/src/lib/rpc.ts`。

### [ ] N1-T03：将 nginx CSP 从 report-only 升级为强制执行

优先级：N1
目标：让内容安全策略真正阻断 XSS 注入向量，而非只上报。

- [ ] 在本地开发环境打开浏览器 console，收集 `Content-Security-Policy-Report-Only` 当前上报的所有违规来源（script/style/img/connect/font）。

- [ ] 在 `apps/web/nginx.conf` 中将 `Content-Security-Policy-Report-Only` 改为 `Content-Security-Policy`，保留 `report-uri` 或 `report-to` 供后续监控。
- [ ] 本地 `docker compose up` 验证前端加载无 CSP 阻断报错。
- [ ] 影响文件：`apps/web/nginx.conf`。
