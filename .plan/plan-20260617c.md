## 問題清單

| #    | 嚴重度 | 檔案                          | 問題                                                                                                                                                     |

| C1-1 | HIGH   | `apps/web/src/lib/push.ts:75` | B2-1 catch unsubscribe() 後直接呼叫 subscribe()，當瀏覽器保留舊訂閱時 subscribe({newKey}) 拋 InvalidStateError（W3C §4.3，Chrome/Firefox/Safari 均實施） |
| C1-2 | LOW    | `apps/web/public/sw.js:65`    | 靜態資源 cache-first handler 在 cache miss 時 fetch() 無 try/catch，離線時拋未處理錯誤                                                                   |

## 修復方案

### C1-1（HIGH）— push.ts

在 catch 區塊內新增 `reg.pushManager.getSubscription()` 重新查詢。
若訂閱仍存在（瀏覽器保留本地記錄）→ 拋出清晰的 `push-rotation-blocked` 領域錯誤；
若訂閱已消失（部分實作在 throw 時仍清除記錄）→ 繼續執行 subscribe()。

```ts
} catch {
    const stillActive = await reg.pushManager.getSubscription();
    if (stillActive) throw new Error("push-rotation-blocked");
}
```

SettingsPage.tsx 的既有 `else { toast(t("settings.pushFailed")) }` 已涵蓋此錯誤。

### C1-2（LOW）— sw.js

```js
const res = await fetch(e.request).catch(() => null);
if (!res) return Response.error();
```
