# Plan 2026-07-02 — 自动删除：取消拆分（永不/推迟）+ 总开关

## 背景

生产巡检发现两个 P0：

1. 队列条目的"取消"只删队列行，次日 Phase 1 会重新入队——取消形同虚设。
2. 定时任务的"暂停"（手动移除 repeatable job）撑不过 API 重启，`startScheduler()` 会重新注册。

另有 P0（2026-07-02 已诊断）：Phase 2 的 16/17 条 500 = Jellyfin 10.11 UserData 墓碑
唯一约束 bug——文件删除**成功后**更新 SQLite UserData 表失败返回 500，Jellyfin DB 留下
幽灵条目骗过存在性检查。用户已确认磁盘文件实际全部删除。同日发现 AncestorIds 被
10.11 /Items 静默忽略、AnyProviderIdEquals 返回未过滤结果——库范围保护改用逐库
ParentId + 客户端 TMDB 匹配（同轮修复）。

## 用户已确认的决策

| 决策点       | 结论                                                        |
| ------------ | ----------------------------------------------------------- |
| 推迟删除时长 | 固定 7 天，到期自动重新进入两段式（重新标记 → 次日删除）    |
| 永不删除粒度 | 按条目：季度条目只排除该季；整剧条目排除整部剧              |
| 排除管理入口 | 设置页排除列表（可移除）+ 剧集详情页"永不自动删除"开关      |
| 本轮附加范围 | 自动删除总开关（默认关闭）；其余项写入 requirement 后续规划 |

## 推导出的默认规则（用户可否决）

- **季度排除阻断整剧删除**：某季被"永不删除"或处于推迟期时，该剧不允许整剧入队
  （否则删整剧会连带删掉被保护的季，违背保护语义）。
- **卡片角标上的 X = 推迟 7 天**（软默认，误触可恢复）；"永不删除"只在设置页队列
  面板和详情页开关提供（避免小角标塞两个按钮）。
- **总开关默认 OFF**：迁移后所有用户自动删除停用，需在设置页手动打开——同时解决
  "暂停撑不过重启"问题（cron 恢复注册也安全，job 开头检查开关）。
- **创建排除时同步清理队列**：详情页开关或队列按钮创建排除时，删除该排除所覆盖的
  现存队列条目（否则已入队条目仍会被 Phase 2 删除）。
- **过期 defer 行由每日 job 顺手清理**。
