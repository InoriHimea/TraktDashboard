## 修復方案

### D1-1 — 新增 i18n key + SettingsPage catch 分支

`zh-CN.json` / `en-US.json` 增加 `pushRotationBlocked` 翻譯。  
`SettingsPage.tsx` catch 加 `else if (err.message === "push-rotation-blocked")` 分支。

### D1-2 + D1-3 — stillActive key 比對（同一修復）

在 `if (stillActive)` 判斷內，先比對 `stillActive.options.applicationServerKey` 與 `keyBytes`：

- 若吻合（D1-2 多分頁競爭 / D1-3 Firefox false-positive）→ 直接以 `stillActive` 呼叫後端並 `return`
- 若不吻合（真正卡住）→ 拋 `push-rotation-blocked`

### D1-4 — getSubscription() 包裹 try/catch

```ts
let stillActive: PushSubscription | null = null;
try {
    stillActive = await reg.pushManager.getSubscription();
} catch {
    // 降級狀態：視為已清除
}
```
