# Plan: 收藏集数支持（Collection Episode-Level）

**日期**: 2026-06-22  
**版本**: v0.69.x  
**来源**: 用户反馈「Trakt collection 按集记录格式，但系统未展示集级概念」

---

## 背景与目标

Trakt Collection API（`/sync/collection/shows?extended=metadata`）返回的数据结构是：

```
剧集 → seasons[] → episodes[] → metadata { media_type, resolution, hdr, audio, audio_channels }
```

v0.68.0 已加上 `?extended=metadata`，但 sync 仍用「首集元数据代替整剧」存一条 show-level 行。  
目标：**在保持现有 show 卡片网格不变的前提下，新增集级元数据的存储与展示**。

---

## 系统现状分析

### DB 层（已就绪）

`user_collection` 表已有 `season`、`episode` 列和双级唯一索引：

- `user_collection_show_idx`：`(userId, showId)` WHERE `season IS NULL AND episode IS NULL`
- `user_collection_episode_idx`：`(userId, showId, season, episode)` WHERE 两者 `IS NOT NULL`

可同时存 show-level（grid 卡片用）和 episode-level（详情用），**无需 schema 变更**。

### 路由层（现有逻辑无需改动）

- `GET /collection`：`isNull(season)` 过滤，只返回 show-level → 卡片网格，**不变**
- `GET /collection/check`：用 `isNull(season)` 判断是否收藏 → **不变**
- `DELETE /collection/:id`：已有注释"episode-level rows 不 propagate 到 Trakt" → **不变**
- `POST /collection/clear-remote`：用 show-level rows 构建删除列表 → **不变**

## 实现计划

### T01 — sync.ts：追加 episode-level 行 `[fix]`

**文件**: `apps/api/src/services/sync.ts`  
**函数**: `syncUserCollection()`，show sync 块（约第 2013 行）

现有逻辑：upsert 一条 show-level 行（`season=NULL, episode=NULL`），format 取首集元数据。  
新增逻辑：**在 upsert show-level 行之后**，遍历 `ts.seasons[].episodes[]`，每集 upsert 一条 episode-level 行。

```typescript
// After the show-level upsert:
for (const season of ts.seasons ?? []) {
    for (const ep of season.episodes ?? []) {
        const epMeta = ep.metadata;
        const epCollectedAt = ep.collected_at ? new Date(ep.collected_at) : collectedAt;

        const [existingEp] = await db
            .select({ id: userCollection.id })
            .from(userCollection)
            .where(
                and(
                    eq(userCollection.userId, userId),
                    eq(userCollection.showId, localId),
                    eq(userCollection.season, season.number),
                    eq(userCollection.episode, ep.number),
                ),
            )
            .limit(1);

        const epVals = {
            mediaFormat: epMeta?.media_type ?? null,
            resolution: epMeta?.resolution ?? null,
            hdr: epMeta?.hdr ?? null,
            audio: epMeta?.audio ?? null,
            audioChannels: epMeta?.audio_channels ?? null,
            collectedAt: epCollectedAt,
            updatedAt: now,
        };

        if (existingEp) {
            await db.update(userCollection).set(epVals).where(eq(userCollection.id, existingEp.id));
        } else {
            await db.insert(userCollection).values({
                userId,
                mediaType: "episode",
                showId: localId,
                season: season.number,
                episode: ep.number,
                ...epVals,
                createdAt: now,
            });
        }
    }
}
```

**验收**：同步后 `SELECT COUNT(*) FROM user_collection WHERE season IS NOT NULL` > 0。

### T02 — collection.ts：新增集详情端点 `[feat]`

**文件**: `apps/api/src/routes/collection.ts`

新增路由 `GET /api/collection/shows/:showId/episodes`：

```typescript
// GET /api/collection/shows/:showId/episodes
// 返回该剧的所有 episode-level 行，按 season+episode 排序，分组为季
collectionRoutes.get("/shows/:showId/episodes", async (c) => {
    const userId = c.get("userId");
    const showId = Number(c.req.param("showId"));
    if (!Number.isFinite(showId)) return c.json({ error: "Invalid showId" }, 400);
    const db = getDb();

    const rows = await db
        .select()
        .from(userCollection)
        .where(
            and(
                eq(userCollection.userId, userId),
                eq(userCollection.showId, showId),
                isNotNull(userCollection.season),
                isNotNull(userCollection.episode),
            ),
        )
        .orderBy(userCollection.season, userCollection.episode);

    // Group by season
    const seasons = rows.reduce(
        (acc, r) => {
            const s = r.season!;
            if (!acc[s]) acc[s] = [];
            acc[s].push({
                episode: r.episode!,
                mediaFormat: r.mediaFormat,
                resolution: r.resolution,
                hdr: r.hdr,
                audio: r.audio,
                audioChannels: r.audioChannels,
                collectedAt: r.collectedAt?.toISOString() ?? null,
            });
            return acc;
        },
        {} as Record<number, CollectionEpisodeDetail[]>,
    );

    return c.json({ data: seasons });
});
```

**验收**：`GET /api/collection/shows/123/episodes` 返回 `{ data: { "1": [{episode:1,...}, ...], "2": [...] } }`。

```typescript
export interface CollectionEpisodeDetail {
    episode: number;
    mediaFormat: string | null;
    resolution: string | null;
    hdr: string | null;
    audio: string | null;
    audioChannels: string | null;
    collectedAt: string | null;
}

export interface CollectionShowEpisodes {
    // key = season number (string, since JSON object keys are strings)
    [season: string]: CollectionEpisodeDetail[];
}
```

### T05 — CollectionPage.tsx：Show 卡片 + 集详情面板 `[feat]`

**原则**: 保持现有网格布局，新增集级细节展示，不破坏现有导航。

**改动点 1 — Show 卡片底部增加集数触发器**

在 `CollectionCard` 的 `item.year` 下方，对于 show 类型且有 episode-level 行时（通过卡片外 prop 传入 episodeCount），显示集数 badge：

```tsx
{
    isShow && episodeCount > 0 && (
        <button
            onClick={(e) => {
                e.preventDefault();
                onExpand?.();
            }}
            style={
                {
                    /* 紧凑小按钮 */
                }
            }
        >
            {episodeCount} 集收藏 ›
        </button>
    );
}
```

**改动点 2 — 新增 `CollectionEpisodePanel` 组件（模态框）**

点击「N 集收藏 ›」→ 打开全屏半透明模态，展示：

- 顶部：剧名 + 海报缩略图
- 正文：按季分组的集列表
    - 季标题（`第 N 季`）
    - 每集一行：`S01E01` + 格式角标（`4K · DV · Blu-ray · DTS-X`）+ 收藏时间

格式角标复用现有 `formatBadge()` 函数（或提取为独立工具函数）。

**改动点 3 — 主列表查询附加 episodeCount**

修改 `GET /api/collection`，在 show-level 行旁附带该 show 的 episode 行数量：

```sql
-- 子查询附带集数
SELECT col.*,
    (SELECT COUNT(*) FROM user_collection ep
     WHERE ep.user_id = col.user_id AND ep.show_id = col.show_id AND ep.season IS NOT NULL) AS episode_count
FROM user_collection col ...
```

或直接在前端调用时按需请求（懒加载，点击才获取集列表）。

**推荐策略：懒加载**

- 主列表仅查 show-level 行（如现在），不附带集数查询
- 集数通过点击触发请求（`useCollectionShowEpisodes(showId)` 在 modal 内触发）
- 首屏渲染性能不受影响
