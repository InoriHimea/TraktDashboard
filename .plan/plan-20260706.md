# Plan 2026-07-06 — N6 项目深化（分四批）

## 背景

N5 完成后的健康评估发现的真实缺口（详见会话内探索结论）：

1. **自动删除实际不可启用**：Jellyfin 10.11 的 UserData 墓碑 bug（jellyfin#16975）使
   播放过的内容删除必 500；且 Phase 2 失败后队列行被删、次日 Phase 1 重新入队 → 每日
   重试死循环。N5-T10 是最后一公里。
2. **最危险的代码零测试**：jellyfin-auto-delete.ts 是全仓库唯一真删文件的代码，无单测。
3. **CI 红灯无人看见**：CI 跑在 GitHub（镜像），主远端是自建 git 服务器；web 测试红了
   数周才被发现。通知渠道用户指定 Gotify（https://gotify.mellivora.com.cn）。
4. **备份只写不读**：4 个 provider 均无 download 函数、无恢复端点；且用户报告
   GDrive/OneDrive 当前"貌似不可用"。
5. **路线图腐化**：N1-T01/T03 代码早已完成、N2-T01/T02、F04–F12、收藏集元数据 T01–T05
   全部已发货，但文档标"待做/规划中"。
6. **移动端适配未审计**：设置页等固定宽度双列布局，窄屏可能溢出。
