## F13 · Collection 本地同步（v0.62.0）

Trakt 免费版限 100 条收藏。策略：全量拉取 → 写本地 DB → 删除远端 → 本地为 source of truth。

### 设计要点

- Trakt Collection API：`GET /sync/collection/shows|movies`、`POST /sync/collection`、`POST /sync/collection/remove`
- 元数据：媒体格式（blu-ray/digital/dvd）、分辨率（uhd_4k/hd_1080p）、HDR、音频
- 本地 `user_collection` 表存储后，执行"清空远端"操作（用户手动确认）
- CollectionPage 展示本地收藏，支持过滤 / 按格式分类

## F14 · 云端 DB 备份（v0.63.0）

将本地 PostgreSQL 数据导出并备份到云存储，支持恢复。

### 设计要点

- **Google Drive**：OAuth2 Device Flow（`POST /api/backup/google/auth` 启动，轮询 token），`googleapis` SDK 上传 pg_dump gzip 文件
- **WebDAV**：支持 Nextcloud / 坚果云 / 任意 WebDAV，用账号密码配置，无需 OAuth
- 备份内容：`pg_dump -Fc` 格式（可恢复），文件名 `trakt-dash-backup-YYYYMMDD-HHmmss.pgdump.gz`
- 定时：可选每日自动备份（cron job in scheduler）
- 恢复：上传备份文件 → `pg_restore`（需用户确认，破坏性操作）
- SettingsPage 新增 "备份与恢复" 区块

### 任务清单

- [ ] M1 `settings` 表扩展 — `gdrive_token`、`webdav_url/user/pass` 字段（加密存储）
- [ ] M2 `services/backup.ts` — pg_dump + gzip + 上传逻辑
- [ ] M3 Google Drive OAuth Device Flow 路由
- [ ] M4 WebDAV 上传（用 `webdav` 包）
- [ ] M5 `routes/backup.ts` — trigger/status/list/restore
- [ ] M6 SettingsPage 备份区块 UI
- [ ] M7 i18n + TypeScript + v0.63.0
